Closed Loop: Badge

Fixed after proof

Badge was the named holdout from the first proof pass. The follow-up closed it by adding stable intent/risk attributes in the wrapper and mapping those states through badge.* component tokens in the Mantine theme.

The important part is the loop: the proof named a real drift, the fix changed the runtime path, and painted DOM parity verified the result. The remaining question is API shape: keep arbitrary Mantine colors, or expose only named Clementine intents.
