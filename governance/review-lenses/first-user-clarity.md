# First-User Clarity Lens

Use for public demos, docs entry points, onboarding, and the Observatory playground.

## Check

| Area | Clementine expectation |
|---|---|
| 3-second read | A first-time viewer can tell Clementine is a design system with machine-readable contracts. |
| Product signal | The first viewport shows actual product/system evidence, not only atmosphere. |
| Primary action | One action is visually primary and names what happens next. |
| Vocabulary | Agentic terms are paired with concrete system proof. |
| Navigation | A user can reach proof, playground, governance, and docs without guessing. |
| Visual hierarchy | Brand expression supports comprehension instead of delaying it. |

## Clementine-Specific Red Flags

- "AI-ready" appears without proof of specs, tokens, validators, or parity.
- Visuals show a generic design system but not the governance machinery.
- The playground looks like marketing only, with no live component or contract evidence.
- The page assumes the viewer already understands the 9-tray architecture.
